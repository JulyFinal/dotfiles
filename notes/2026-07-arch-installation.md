# 2026 年 7 月 Arch 装机复盘

最后更新：2026-07-22

这次安装的目标逐渐从做一套完整的桌面美化，变成保留真正方便复用的部分。现在的仓库负责开发环境、编辑器、终端、启动器、代理服务和软件清单。Niri、输入法皮肤、状态栏、锁屏、通知样式和壁纸仍留在本机，等用法稳定后再决定是否管理。

## 现在留下的结构

Chezmoi 管理用户配置，Pacman 和 Paru 清单记录 Arch 软件，Mise 管理开发工具版本。`bootstrap` 负责新机器初始化，`doctor` 和 `tests/pre-commit` 用来检查运行状态、敏感信息和仓库结构。

代理最后切到了 Mihomo。它作为用户服务常驻，在 `127.0.0.1:10808` 提供 HTTP 和 SOCKS 混合端口。订阅、OpenVPN 参数和密钥只保存在本机私有文件中，不进 Git。需要代理的程序通过环境变量或单独的启动命令接入，不再依赖一个长期打开的桌面窗口。

Vicinae 留作应用启动器和剪贴板历史。编辑器以 Neovim 为主，同时保留 Helix 配置，开发工具交给 Mise 安装。桌面层使用 Niri，但目前不把它的配置当成可复制到所有机器的稳定方案。

## 这次遇到的取舍

V2RayA 对这次订阅里的 VLESS XHTTP 节点支持不合适。v2rayN 能连接这些节点，但后台常驻和桌面集成不够省心。Mihomo 跑通后，代理核心不再依赖图形界面，维护起来更简单。

企业微信先后试过 Distrobox、Wine 和远程 Windows。前两种方案出现过重复窗口、黑框、托盘菜单错位和输入法候选框异常，最后不再把这部分塞进本机桌面环境。需要时直接通过远程桌面使用 Windows，相关脚本和容器配置也没有继续放在 dotfiles 里。

桌面美化花了不少时间，但视觉效果和可复用性不是一回事。Waybar、Quickshell、Fcitx5 皮肤、greetd 和锁屏配置都适合边用边改，目前不值得让 `bootstrap` 接管。仓库只保留下一台机器确实需要恢复的内容。

## 下次装机时按这个顺序

1. 安装基础系统、网络、蓝牙和 Git，再克隆 dotfiles。
2. 用 `bootstrap` 安装通用软件和 Mise 工具，运行 `doctor` 检查基础环境。
3. 单独恢复代理订阅、OpenVPN 参数、密钥和设备配对，不从 Git 获取这些内容。
4. 安装桌面环境后先用默认配置，确认窗口、输入法、通知和多显示器都正常，再逐项美化。
5. 用一段时间后再决定哪些桌面配置值得加入 Chezmoi，临时实验继续留在本机。

提交前运行 `tests/pre-commit worktree`。它会检查敏感信息、文件格式和 Chezmoi 渲染结果，避免把本机私有状态带进仓库。
